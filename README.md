## 🚀 Projects

### 🌟 1. Production Legal NLP System: Clause Classification & Generation
* **What It Does**: Automates contract analysis for law firms and compliance teams by categorizing clause segments into distinct categories and dynamically generating legal provisions.
* **System Architecture & AI Flow**:
    ```
    Raw Contract Data ──> Ingestion & Preprocessing ──> Legal-BERT (Classification) ──> LIME Audit ──> FastAPI/Docker
                                                    ──> T5 Fine-Tuning (Generation) ──┘
    ```
* **Core Features**:
    * Fine-tuned a domain-specific **Legal-BERT Transformer** achieving an **F1-score of 0.85** on test sets (a 12 pp increase over baseline BERT architectures).
    * Integrated a generative **T5 Transformer model** to construct correct contractual clauses, reducing document review times by **70%**.
    * Embedded **LIME (Local Interpretable Model-agnostic Explanations)** to trace prediction tokens, guaranteeing audit transparency for corporate legal reviews.
    * Automated deep pipeline tuning across learning-rate bounds via **Optuna (TPE Sampler)** across 50 extensive iterations.
    * Packaged inference wrappers via **FastAPI** hosted inside a containerized **Docker environment**, maintaining **sub-200 ms p95 low-latency serving**.
* **Tech Stack**: Legal-BERT • T5 • PyTorch • LIME • Optuna • FastAPI • Docker • Python

### 🌟 2. Algorithmic Bias Detection & Mitigation Framework
* **What It Does**: Evaluates financial lending workflows to discover systematic algorithmic biases against demographic sub-groups and actively transforms the underlying training representations to enforce fairness guarantees.
* **System Architecture & AI Flow**:
    ```
    Raw Loan Dataset ──> Bias Auditing (AIF360) ──> Pre-processing Reweighing ──> Fair Model Training ──> Strategic Evaluation
    ```
* **Core Features**:
    * Evaluated structural dataset layers discovering pre-mitigation bias markers against protected gender parameters (**Disparate Impact of 0.78**; below the 0.80 EEOC ceiling).
    * Injected **AIF360 mitigation modules** (Reweighing and Disparate Impact Removers) onto incoming features prior to model optimization.
    * Successfully shifted **Disparate Impact to 0.97 (a +24.6% improvement)** and scaled down Statistical Parity Difference to **-0.02 (an 88.2% drop)** while maintaining an exceptional **classification F1-score of 0.82**.
* **Tech Stack**: AIF360 • Scikit-Learn • Logistic Regression • Bias Metrics Suite • Python • Matplotlib

### 🌟 3. Hotel Booking Demand Forecasting & Revenue Optimization
* **What It Does**: Predicts client cancellations, builds customer lifetime profiles, and calculates dynamic pricing recommendations.
* **Core Features**:
    * Optimized a unified machine learning environment benchmarking 5 predictive frameworks (XGBoost, Random Forests, Neural Networks) targeting multi-class labels.
    * Built structured pipeline transformations incorporating Chi-Square feature dependency validation and Variance Inflation Factor (VIF) checks, **trimming dimensionality by 35%**.
    * Utilized **SMOTE** techniques resolving a **14:1 class imbalance** to elevate minority cancellation class recall from **44% to 77%**.
    * Engineered high-yield **RFM customer profiling mixed with K-Means clustering**, capturing a premium customer layer (8% of users) producing **31% of gross revenue**.
* **Tech Stack**: XGBoost • Random Forest • TensorFlow/Keras • SMOTE • Scikit-Learn • PCA • Time-Series Decomposition

* **Core Features**:
